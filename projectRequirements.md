# 项目需求说明

## 项目目的

本项目的目的是通过对家庭局域网中的网络设备接入与断开的监控与识别，从而及时通知我家人回家和离家的消息。

## 技术架构

+ 项目主要采取python语言加docker化的部署模式，整体是部署在群晖NAS中

## 技术方案要点

+ 项目可以使用 Python 原生 ARPING 探测局域网内的设备清单，并周期性的比对设备清单的变化
+ 应该支持一个本地化的文件去配置设备MAC与设备名称的映射关系
+ 通知使用bark api进行，api key是在本地化的文件中去配置
+ 网络里的设备种类有很多种，应该支持在文件中配置，哪一个设备无需触发通知，哪个设备要触发bark震动级别的通知或者静默级别的通知

## 其他要求

+ 项目在本地调试时应使用python虚拟环境，如果虚拟环境不存在则自行创建
+ 项目应提供docker部署文档 以及 readme文档说明配置项
+ 项目初始化为git项目，并在完成开发后推送到GitHub中
